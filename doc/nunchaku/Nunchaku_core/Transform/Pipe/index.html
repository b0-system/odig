type ('a, 'b, 'c, 'd) t = private
| Id : ('a, 'a, 'c, 'c) t

no transformation

| Fail : ('a, 'b, 'c, 'd) t

yields empty list

| Flatten : ('a, 'b list, 'c, 'd) t -> ('a, 'b, 'c, 'd) t
| Close : ('b1 -> ('c1 -> 'd) -> 'b2 * ('c2 -> 'd)) * ('a, 'b1, 'c1, 'd) t -> ('a, 'b2, 'c2, 'd) t
| Comp : ('a, 'b, 'e, 'f) transformation * ('b, 'c, 'd, 'e) t -> ('a, 'c, 'd, 'f) t
| Fork : ('a, 'b, 'c, 'd) t * ('a, 'b, 'c, 'd) t -> ('a, 'b, 'c, 'd) t

Composite transformation from 'a to 'b, with a reverse transformation from 'c to 'd

Every constructor from here is "smart":

  • Fail is right-absorbing (e.g. compose f fail = fail)
  • Fork with Id in both branches becomes the Id
  • Fork with Fail in a branch becomes the other branch
